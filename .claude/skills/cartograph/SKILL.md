# Cartograph — Semantic Graph Query Engine

Cartograph builds and queries a semantic graph from AID skeleton files. It answers structural questions about the codebase without parsing source code.

## Prerequisites

- `.aidocs/` directory with AID skeleton files (generated by `aid-gen-go` or `aid-gen-py`)
- Cartograph binary (build from source or install)

## Auto-Discovery

Cartograph automatically walks up from the current directory to find `.aidocs/`. You only need `--dir` when the AID files are in a non-standard location.

## Graph Cache

On first run, cartograph creates a `cartograph.cache` file alongside the AID files. Subsequent runs load the cached graph (~6x faster). The cache auto-invalidates when any AID file changes. Use `--no-cache` to bypass.

## Commands

### Find callers of a function (trace up)
```bash
cartograph callstack <function> --up --format json --dir .aidocs/
```

### Find callees of a function (trace down)
```bash
cartograph callstack <function> --down --format json --dir .aidocs/
```

### Find callers and callees (both directions)
```bash
cartograph callstack <function> --both --format json --dir .aidocs/
```

### Find all dependents of a type
```bash
cartograph depends <type> --format json --dir .aidocs/
```

### Find what produces an error
```bash
cartograph errors <ErrorType> --format json --dir .aidocs/
```

### Find what touches a field
```bash
cartograph field <Type.Field> --format json --dir .aidocs/
```

### Trace side effects of a function
```bash
cartograph effects <function> --format json --dir .aidocs/
```

### Search for nodes by name (glob or regex)
```bash
cartograph search <pattern> --dir .aidocs/
cartograph search "*Handler*" --kind Type --dir .aidocs/
```

### List all nodes in a module
```bash
cartograph list <module> --dir .aidocs/
```

### Show graph statistics
```bash
cartograph stats --dir .aidocs/
```

### Method-qualified queries
For methods, use `Type.Method` syntax:
```bash
cartograph callstack WALManager.Close --up --format json --dir .aidocs/
```

## Flags

| Flag | Description |
|------|-------------|
| `--dir <path>` | Path to .aid files directory (default: auto-discover `.aidocs/`) |
| `--format <fmt>` | Output format: `tree` (default), `json` |
| `--depth <n>` | Max traversal depth, 1-50 (default: 10) |
| `--no-cache` | Disable graph cache (always re-parse AID files) |

## Output Format

JSON output contains:
- `query` — what was asked
- `summary` — human-readable summary
- `node_count` — number of nodes in result
- `paths` — array of paths, each with `nodes` and `edges`

Each node has:
- `name` — short name (e.g., "Close")
- `qualified_name` — full name (e.g., "journal.WALManager.Close")
- `kind` — "Function", "Method", "Type", "Field", "Constant", "Trait", "Module", "Workflow", "Lock"
- `module` — package name
- `source_file` — relative path to source file
- `source_line` — line number of definition

## Tips

- Use `--format json` for programmatic consumption, omit for human-readable tree output
- Auto-discovery means you rarely need `--dir` — just run from within the project
- Results only include symbols present in AID skeletons — use `aid-gen-go --internal` for unexported function coverage
- The `search` command supports glob patterns (`*Handler*`) and regex
- Use `stats` to quickly verify the graph loaded correctly
